Security + Auth

Đăng nhập đa phương thức và bảo vệ phiên người dùng

Hệ thống hỗ trợ local account + Google OAuth2, có cơ chế khóa tài khoản, token CSRF và interceptor kiểm soát truy cập.

Luồng xác thực

B1

Login/signup qua form tại /login, /signup.

B2

Xác thực PasswordEncoder BCrypt.

B3

Tạo session currentUser và rotate CSRF token.

B4

Admin chuyển /admin, user về trang chủ.

Google OAuth2
  • Đăng nhập bằng Gmail, tự tạo user nếu chưa tồn tại.
  • Cho phép liên kết Gmail với tài khoản hiện có tại trang profile.
  • Lưu trạng thái gmailVerified, googleSub, googleAvatarUrl.
AuthController, AuthService, SecurityConfig

Cơ chế an toàn

  • Tự khóa 15 phút sau 5 lần nhập sai mật khẩu liên tiếp.
  • Admin khóa theo 1/3/7/14/30 ngày hoặc vĩnh viễn + lý do.
  • AuthCsrfInterceptor kiểm tra token cho mọi POST (trừ webhook SePay).
  • LockedAccountInterceptor tự logout user bị khóa trong phiên đang mở.

Payment integration

Webhook SePay xác nhận chuyển khoản và cập nhật đơn tự động

Hệ thống xác thực API key, kiểm tra idempotent giao dịch, đối chiếu số tiền và tự cập nhật trạng thái đơn hàng.

Step 1

Nhận POST /webhooks/sepay với header Authorization: Apikey ...

Step 2

Validate transferType=in và tìm orderCode trong payload.

Step 3

Kiểm tra giao dịch đã xử lý chưa qua paymentTransactionId.

Step 4

Đối chiếu số tiền chuyển khoản = totalAmount đơn hàng.

Step 5

Set paymentConfirmed=true, lưu gateway/ref/transactionId.

Step 6

Đổi trạng thái đơn sang DELIVERED, log audit, gửi notification.

Điểm bảo vệ quan trọng

  • Từ chối nếu API key sai hoặc thiếu.
  • Từ chối giao dịch không phải tiền vào.
  • Từ chối nếu đơn hết hạn thanh toán.
  • Từ chối nếu số tiền không khớp đơn.

Tự động sau xác nhận

  • Refresh rank người dùng theo đơn đã giao.
  • Gửi thông báo user + admin loại ORDER_BANK_PAID.
  • Ghi audit action: SEPAY_WEBHOOK_PAID.
